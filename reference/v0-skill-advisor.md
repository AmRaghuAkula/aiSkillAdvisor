## Skill Advisor

**Product identity.** This is the v0 of the **Skill Advisor** — a discipline + algorithm that helps non-technical builders proactively invoke the right skill at the right moment from a 180+ skill inventory. Currently dogfooded on the dogfood project. Vision is to extract as a standalone open-source product on GitHub.

**Purpose.** Surface skill suggestions BEFORE the founder asks. The founder is not technical-leaning; the inventory is 180+ skills; without prompts, the right skill goes uninvoked. The Skill Advisor is the discipline that prevents missed-skill failures.

**TDD-validated 2026-06-01.** Restructured after `superpowers:writing-skills` baseline testing exposed 3 failure modes (anchor bias, trivial-action misfire, closed-world assumption). All counter-rules embedded below.

## RED FLAGS — STOP if you find yourself saying these

If your inner voice is producing any of these phrases, **STOP**. You are about to misroute or loop:

| Verbatim red-flag phrase | Why it's wrong | Correct action |
|---|---|---|
| *"There's no row for that, I'll just lean on what fires"* | Anchor bias (Pattern 1). You found one trigger and stopped scanning. | List ALL work-types this task spans; match each separately. |
| *"It's a small CSS tweak — anti-pattern says don't suggest skills for trivial actions"* | Trivial-action misfire (Pattern 2). Quality-judgment phrasing overrides triviality. | Re-scan founder's request for "premium", "feels like", "polish", "$X/mo feel". If present → surface design skill regardless of LOC. |
| *"This skill is in the inventory but not the trigger table, so I shouldn't surface it"* | Closed-world assumption (Pattern 3). The 17/13 trigger rows are GUIDANCE, not whitelist. | Surface the skill anyway. Propose adding it to the table. |
| *"I'll just fix this quickly while I'm here"* | Loop-creep onset (L4 violation). Scope is drifting. | STOP. Park the unrelated finding. Return to original goal. |
| *"One more skill should do it"* | Invocation budget violation (L2). | STOP. Surface to founder for `--continue-chain` permission. |
| *"It's related so I should chain"* | Loop-creep rationalization. | STOP. The chain itself is the warning. |

## Loop-Prevention Layer (L1–L5)

These apply across ALL work-types and sections. They prevent the cascading-remediation failure mode (fix A → break B → fix B → break C → loop).

| ID | Mechanism | Rule |
|---|---|---|
| **L1** | **Goal anchor** | At session start, declare the ONE goal in one sentence. Pinned at top of context. Every skill invocation must demonstrably serve it. |
| **L2** | **Invocation budget** | Max **2 MUTATING skill invocations per single goal**. Third mutating invocation requires explicit founder permission. **EXCEPTION:** read-only/planning skills do NOT count against budget — `brainstorming`, `spec`, `writing-plans`, `code-review`, `design-review`, `verification-before-completion`, `investigate`, `browse`, `qa-only`, `health`. Only mutating/deploying skills count: `ship`, `deploy`, `careful`, `freeze`, `autoplan` (when it edits), migrations, etc. |
| **L3** | **Regression detection** | Each invocation declares preconditions ("what currently works") + postconditions ("what should work after"). If a previously-passing precondition now fails → **immediate rollback**, NOT chain forward. |
| **L4** | **Scope-creep alarm** | Track files touched per session. If file-set grows beyond ~8 files OR crosses a section boundary mid-session → pause and surface. |
| **L5** | **Cycle detection** | If same skill invoked twice for same goal OR pattern A→B→A emerges → **abort and surface**. Don't invoke a third. |

### Routing cases

| Case | When | Action |
|---|---|---|
| **A** | ≥80% signals align with one section | Auto-route silently. No notification. |
| **B** | Work spans §1 + §2 (cross-section) | Surface ONCE at session start: "This crosses §1 + §2. Anchoring at §X for bulk; will flag §Y crossover." |
| **C** | Truly novel domain (≤30% signal match) | Surface ONCE: "(a) Use §1+§2 as proxy, (b) propose §N, (c) skip advisor for this session." 30s default → (c). |
| **D** | Same-session pivot detected | Auto-detect via file-watchlist. Quietly switch section. |
| **E** | Multi-type work | Surface ALL applicable skills. **CASE E supersedes CASE B when both apply.** |
| **F** | Read-only / exploration / Q&A | Skip advisor entirely. |

## Format Conventions

When surfacing, use this exact format:

> **Skill suggestion:** `<skill-name>` — `<what it would do here>`. Want me to invoke it?

For risk moments:

> **Risk flag:** about to `<destructive action>`. `<skill-name>` would `<safety mechanism>`. Recommend invoking first.

| Pattern | Verbatim rationalization | Counter-rule |

## Open-World Rule

The skill inventory is **180+ skills** across multiple plugin ecosystems. The trigger tables in this file are **GUIDANCE, not an exhaustive whitelist**.

When you encounter work that fits a skill outside the trigger table:
1. Surface the skill anyway
2. Note: "Skill not in trigger table — proposing addition: <work-type> → `<skill-name>`"
3. If accepted, the trigger gets added in the next edit cycle

---

## Sunset Condition

If the user repeatedly declines suggestions over 2+ sessions OR explicitly says "stop suggesting skills":
- Dial down to skill suggestions only when a CRITICAL risk applies
- Continue Loop-Prevention Layer (L1-L5) regardless — those are safety, not advisory
